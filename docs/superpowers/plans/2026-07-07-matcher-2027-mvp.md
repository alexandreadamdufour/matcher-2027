# matcher-2027 MVP Implementation Plan

> Exécution : directe, inline, sans dispatch de subagents ni checkpoints d'approbation —
> l'utilisateur a explicitement demandé d'enchaîner sans validation intermédiaire (budget 3h,
> dégrader le scope si blocage > 10 min). Plan condensé (task list + file map) plutôt que
> bite-sized TDD complet, pour rester dans le budget.

**Goal:** Ossature technique déployée de matcher-2027 (Next.js 15 + PWA) avec données seed
fictives (4 candidats, 30 thèses), calcul d'affinité 100% client, pages landing/questionnaire/
résultats/partage/méthodologie/sources, repo GitHub public + déploiement Vercel.

**Architecture:** App Router, données statiques JSON validées par Zod dans `/content`, state du
questionnaire en mémoire (React state, persist en sessionStorage pour survivre au refresh),
scoring pur (fonction testée unitairement), rendu des résultats côté client uniquement.

**Tech Stack:** Next.js 15, TypeScript strict, Tailwind, shadcn/ui, Zod, Vitest (test unitaire
scoring), pnpm.

## Global Constraints

- Aucune mention de vrais partis/candidats/personnalités (code + seed data)
- 100% du calcul côté client, aucune donnée envoyée à un serveur
- Pas de DB, pas d'auth, pas d'analytics, pas de scraping, pas de tests E2E
- Mobile-first, testé à 375px ; accessibilité clavier/ARIA/contraste AA
- 30 thèses / 6 catégories (Économie, International, Écologie, Institutions, Société, Éducation)
- 4 candidats fictifs : Alpha, Beta, Gamma, Delta

## Tasks

### Task 1 — Scaffold Next.js
- `pnpm dlx create-next-app@latest` (TS strict, Tailwind, App Router, src dir, pnpm)
- Installer shadcn/ui (init + composants: button, card, checkbox, progress)
- Installer zod, vitest, @vitejs/plugin-react
- Configurer `vitest.config.ts` + script `test` dans package.json
- Commit: `chore: scaffold next.js app`

### Task 2 — Schémas Zod + chargement contenu
- `src/lib/schemas.ts` : ThesisSchema, CandidateSchema, PositionSchema, UserAnswerSchema
  (stance enum -2..2 via z.union of literals, weight 0|1|2)
- `src/lib/content.ts` : lit les JSON de `/content`, valide avec les schémas, exporte
  `getTheses()`, `getCandidates()`, `getPositions()`
- Commit: `feat: add zod schemas and content loader`

### Task 3 — Contenu seed
- `content/thesis-guidelines.md`
- `content/theses.json` (30 thèses, 6 catégories, formulation neutre/symétrique)
- `content/candidates/*.json` (Alpha/Beta/Gamma/Delta, fictifs)
- `content/positions/*.json` (positions cohérentes par candidat, source_url/type/quote/date
  fictifs mais réalistes)
- Commit: `content: add seed theses, candidates and positions`

### Task 4 — Moteur de scoring + test
- `src/lib/scoring.ts` : `computeAffinity(answers: UserAnswer[], positions: Position[], theses: Thesis[]): CandidateResult[]`
  - distance normalisée = |userStance - candidateStance| / 4
  - pondération = weight (0 exclu, 1, 2)
  - skip (pas de réponse ou stance absente) ignoré du dénominateur
  - retourne score % par candidat + par catégorie + confidence (ratio thèses répondues/total)
- `src/lib/scoring.test.ts` (Vitest) : cas nominal, cas tout-skip, cas pondération double
- `pnpm test` doit passer
- Commit: `feat: add affinity scoring engine with unit test`

### Task 5 — Store des réponses utilisateur
